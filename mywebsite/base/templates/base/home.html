AWS


O que é?


A Amazon Web Services (AWS) é a plataforma de nuvem mais adotada e mais abrangente do mundo, oferecendo mais de 200 serviços completos de datacenters em todo o mundo. Milhões de clientes, incluindo as startups que crescem mais rápido, as maiores empresas e os maiores órgãos governamentais, estão usando a AWS para reduzir custos, ganhar agilidade e inovar mais rapidamente.


Um pouco de história


As origens da plataforma datam de 2002, quando os primeiros serviços foram testados em fase beta — na época, o nome dessa estrtutura era Amazon.com Web Service. A ideia era experimentar a comercialização de ferramentas e infraestrutura para desenvolvedores. Lançado oficialmente em 2006, Amazon Web Services é um provedor de serviços online para websites ou aplicações cliente servidor baseado nas nuvens. A maioria destes serviços não é acessível pela internet, mas oferecem funcionalidades que outros desenvolvedores podem usar em suas aplicações. Amazon Web Services pode ser acessado usando HTTP, protocolo REST, estilo de arquitetura ou pelo protocolo SOAP. O modelo de cobrança pelos serviços é de acordo com o uso.

Treinamentos e Certificações


As certificações e treinamentos da AWS são fundamentais para capacitar profissionais e organizações em tecnologia de nuvem. Oferecendo uma variedade de certificações reconhecidas globalmente e programas de treinamento abrangentes, a AWS permite que indivíduos e equipes adquiram habilidades essenciais para utilizar seus serviços e recursos com eficácia. Esses recursos são essenciais para quem está começando na nuvem ou buscando avançar na carreira de computação em nuvem.


Treinamentos


AWS re/Start: é um programa de treinamento de desenvolvimento de força de trabalho baseado em coorte que prepara indivíduos para carreiras na nuvem e os conecta a empregadores em potencial. Não é necessário um histórico em tecnologia para se inscrever. O programa é gratuito para o aluno e focado em ajudar pessoas desempregadas ou subempregadas a iniciar uma nova carreira. As organizações colaboradoras do AWS re/Start apoiam grupos sub-representados, minorias, indivíduos deslocados, jovens e muito mais.

Certificações

A AWS oferece dois tipos de certificações que se enquadram em duas categorias principais: Certificações Principais (core) e Certificações Especialistas. As Certificações Principais (core) abrangem três áreas de conhecimento: arquitetura, desenvolvimento e operações, e são oferecidas em três níveis de qualificação: Fundamentos, Associado e Profissional.

Método de Cobrança


Pay as you go em nuvem (PAYG) ou Pay Per Use são termos que definem um método de pagamento por serviços de cloud computing pelo uso. Diferente, portanto, das tradicionais mensalidades e pacotes associados a um conjunto de ferramentas fixo, contratado por usuários de tecnologia.


Principais serviços


Infraestrutura como Serviço (IAAS)

Amazon EC2 (Elastic Compute Cloud): permite criar e executar máquinas virtuais na nuvem

Amazon S3 (Simple Storage Service): serviço de armazenamento em nuvem escalável e altamente disponível

Amazon EBS (Elastic Block Store): fornece armazenamento em nível de bloco para uso com instâncias do Amazon EC2



Plataforma como Serviço (PAAS)

AWS Elastic Beanstalk: permite implantar e gerenciar aplicativos da web e serviços em vários idiomas

AWS Lambda: permite executar código sem servidor em resposta a eventos, com escalabilidade e disponibilidade automáticas

Amazon EKS (Elastic Kubernetes Service): permite executar aplicativos Kubernetes gerenciados na AWS




Software como Serviço (SAAS)

Amazon WorkSpaces: serviço de desktop virtual que permite acessar aplicativos, recursos e dados de qualquer dispositivo

Amazon AppStream 2.0: oferece streaming de aplicativos de desktop da AWS para dispositivos de qualquer natureza





DJANGO




O que é?


O Django é um framework web Python de código aberto, que se destaca por oferecer um ambiente que simplifica a criação de soluções web escaláveis, ao mesmo tempo, em que promove o desenvolvimento rápido e um design limpo, proporcionando ferramentas robustas e eficientes para pessoas desenvolvedoras.


O que é um Framework?


Um framework é um conjunto de ferramentas, bibliotecas e convenções pré-definidas que fornecem uma estrutura para desenvolver software. Ele oferece um esqueleto ou uma base sobre a qual os desenvolvedores podem construir suas aplicações, fornecendo soluções genéricas para problemas comuns. Em suma, um framework simplifica o processo de desenvolvimento ao fornecer funcionalidades prontas e padronizadas, permitindo que os desenvolvedores se concentrem mais na lógica específica da aplicação do que em questões técnicas mais básicas. Isso economiza tempo e esforço, aumentando a eficiência e a consistência do desenvolvimento de software.

Como o Django surgiu?

Ele surgiu originalmente em 2003, quando os desenvolvedores Adrian Holovaty e Simon Willison começaram a usar Python para construir aplicações web no jornal Lawrence Journal-World. Eles perceberam que havia uma quantidade significativa de código que podia ser reutilizado em diferentes projetos, então começaram a criar um conjunto de ferramentas para agilizar o processo de desenvolvimento.


Pra que serve o Django?

O Django é um framework que fornece um conjunto abrangente de ferramentas e componentes pré-construídos que simplifica e acelera a criação de aplicações web robustas, escaláveis e seguras.


    Criação de sites e aplicações web

Arquitetura MVT (Models, View, Templates)


A arquitetura MVT (Model-View-Template) é um padrão de arquitetura de software usado em frameworks de desenvolvimento web, como Django (Python). É uma variação do padrão MVC (Model-View-Controller), com algumas diferenças conceituais.



Models: Um “Model” é a fonte única e definitiva de informações sobre seus dados. Ele contém os campos e comportamentos essenciais dos dados que você está armazenando. É a camada relacionada ao banco de dados.


View: É uma função Python que recebe uma solicitação da Web e retorna uma resposta da Web. Essa resposta pode ser o conteúdo HTML de uma página da Web, ou um redirecionamento, ou um erro 404, ou um documento XML, ou uma imagem… ou qualquer coisa, realmente.


Templates: É a camada de apresentação que define a interface do usuário. Os templates são arquivos HTML que incluem tags e filtros do Django para gerar conteúdo dinâmico. Eles são responsáveis por renderizar os dados fornecidos pelas views e apresentá-los ao usuário final de forma formatada e estilizada.


ORM (Object-Relational Mapping)


É a camada de abstração que facilita a interação com o banco de dados relacional através de objetos Python. Com o ORM do Django, você pode definir modelos de dados usando classes Python, onde cada atributo da classe representa um campo na tabela do banco de dados.

Ele separa a lógica de banco de dados da lógica de negócios. Além disso, ele permite que você trabalhe com dados em um formato nativo de objetos Python, o que é mais intuitivo e fácil de trabalhar.

Conclusão


Em conclusão, a combinação do framework Django e a plataforma de nuvem AWS oferece uma poderosa e flexível infraestrutura para o desenvolvimento e implantação de aplicativos web escaláveis e de alto desempenho.


Ao integrar o Django com a AWS, os desenvolvedores podem aproveitar o poder e a flexibilidade de ambos os recursos. Por exemplo, eles podem implantar seus aplicativos Django em instâncias EC2, usar o Amazon RDS para gerenciar bancos de dados relacionais de forma escalável e confiável, e usar serviços como o Amazon S3 para armazenar arquivos estáticos e mídia.
